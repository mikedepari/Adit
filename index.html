4. Alur Kerja Sistem: Dari Input hingga Rekomendasi

Alur ini menjelaskan bagaimana data pengguna diproses dan diklasifikasikan oleh model K-Nearest Neighbors (KNN) untuk menghasilkan rekomendasi menu yang paling relevan.

Tahap 1: Pengumpulan Data & Feature Engineering

Input mentah dari pengguna (Usia, Berat, Tinggi, Aktivitas, Target Kalori) digunakan untuk menghitung fitur penting seperti IMT (Indeks Massa Tubuh) dan TDEE (Total Daily Energy Expenditure).

Tahap 2: Preprocessing dan Vektorisasi

Data dinormalisasi (dibuat seragam) dan data kategorikal (Omnivora/Vegetarian) diubah menjadi vektor numerik agar bisa dibandingkan secara matematis di ruang fitur.

Tahap 3: Prediksi Model KNN (Core ML Process)

Model KNN memproses fitur yang sudah di-engineer untuk melakukan klasifikasi diet. Ini adalah jantung dari proses rekomendasi:

  1. Penentuan K: Model memilih jumlah tetangga terdekat yang akan dipertimbangkan (misalnya, K=5).
  2. Perhitungan Jarak (Distance Calculation): Jarak antara data input pengguna dengan setiap titik data dalam dataset pelatihan (data menu yang sudah ada) dihitung, seringkali menggunakan Euclidean Distance.
  3. Penentuan Kelas Mayoritas: Model mengidentifikasi K titik data dengan jarak terpendek. Kelas (label diet) yang paling sering muncul di antara K tetangga ini akan ditetapkan sebagai hasil prediksi untuk pengguna.

Tahap 4: Output dan Penyajian

Hasil klasifikasi diet dari model (misal: High-Protein) dicocokkan dengan daftar menu spesifik di database dan ditampilkan kepada pengguna.

1. Model Machine Learning: K-Nearest Neighbors (KNN)

Kami memilih dan mengimplementasikan K-Nearest Neighbors (KNN) sebagai model klasifikasi non-parametrik yang efektif untuk memprediksi kategori diet yang sesuai berdasarkan kedekatan fitur nutrisi.

1. Penjelasan Model KNN (Algoritma Supervised Learning)

KNN mengklasifikasikan input baru berdasarkan mayoritas kelas di antara K tetangga terdekatnya dalam data pelatihan. Fitur-fitur pengguna (TDEE, IMT, dan Preferensi) diplot dalam ruang fitur, dan kedekatannya dengan data menu yang sudah ada (label: High-Protein, Low-Carb, dll.) akan menentukan rekomendasinya.

2. Parameter Utama Model

ParameterDeskripsi & Tujuan
K (Jumlah Tetangga)Nilai integer yang menentukan berapa banyak tetangga terdekat untuk menentukan kelas rekomendasi (disarankan ganjil, misal K=5).
Distance MetricMetode penghitungan jarak antara fitur, umumnya Euclidean Distance.
Features/InputTDEE (Kcal), IMT, Preferensi Diet (Encoded), Tingkat Aktivitas (Encoded).
Output/LabelKelas Prediksi (Jenis Diet): Balanced-Omnivora, High-Protein, Low-Calorie, Vegetarian-Balanced.
